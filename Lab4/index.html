    Лабораторная работа №4 Реализация клиентской части средствами Vue.js

    Студент: Шукалов Андрей Денисович
    Университет: ИТМО
    Группа: К3339

    1. Структура маршрутов

    • /login - Страница входа
    • /register - Страница регистрации
    • /rooms - Управление номерами
    • /clients - Управление клиентами
    • /employees - Управление сотрудниками
    • /schedule - Расписание уборок
    • /profile - Профиль пользователя

    2. Описание интерфейсов

    2.1 Страница входа (Login)

    API запрос: POST /api/auth/token/login/

    Назначение: авторизация пользователя.

    Функции:

    • ввод логина
    • ввод пароля
    • отправка запроса авторизации
    • сохранение токена

    2.2 Страница регистрации (Register)

    API запрос: POST /api/auth/users/

    Назначение: создание нового пользователя.

    Функции:

    • ввод имени пользователя
    • ввод пароля
    • регистрация

    2.3 Страница номеров (Rooms)

    API запросы: GET /api/rooms/, POST /api/rooms/, DELETE /api/rooms/{id}/

    Назначение: управление номерами гостиницы.

    Функции:

    • просмотр списка номеров
    • добавление номера
    • удаление номера

    2.4 Страница клиентов (Clients)

    API запросы: GET /api/clients/, POST /api/clients/, POST /api/clients/{id}/checkout/

    Назначение: управление клиентами гостиницы.

    Функции:

    • просмотр клиентов
    • добавление клиента
    • выселение клиента

    2.5 Страница сотрудников (Employees)

    API запросы: GET /api/Employees/, POST /api/Employees/, POST /api/Employees/{id}/dismiss/

    Назначение: управление сотрудниками.

    Функции:

    • просмотр сотрудников
    • добавление сотрудника
    • увольнение сотрудника

    2.6 Страница расписания уборок (Schedule)

    API запросы: GET /api/cleaning_schedules/, PATCH /api/cleaning_schedules/{id}/

    Назначение: управление расписанием уборок.

    Функции:

    • просмотр расписания
    • назначение сотрудника на уборку

    2.7 Страница профиля (Profile)

    API запросы: GET /api/auth/users/me/, PATCH /api/auth/users/me/, POST /api/auth/users/set_password/

    Назначение: управление данными пользователя.

    Функции:

    • просмотр данных пользователя
    • изменение данных
    • изменение пароля

    3. Авторизация пользователя

    В приложении используется токенная авторизация. После успешного входа сервер возвращает токен auth_token Токен сохраняется в localStorage

    4. Компоненты

    В проекте реализованы компоненты, используемые на различных страницах.

    4.1 RoomCard

    Отображает:

    • номер комнаты
    • статус (занят/свободен)
    • тип комнаты
    • этаж
    • телефон
    • цена
    • кнопки действий

    4.2 ClientCard

    Отображает в зависимости от того высленен клиент или нет

    Если не выселен:

    • имя и фамилия клиента
    • Паспорт
    • Город
    • Дата заезда
    • Дата уезда
    • Номер

    Если выселен:

    • имя и фамилия клиента
    • паспорт
    • город
    • дата заезда
    • дата выселения
    • номер в котором проживал

    4.3 EmployeeCard

    Отображает в зависимости от того уволен сотрудник или нет

    Если сотрудник не уволен:

    • ФИО сотрудника
    • уволен/работает
    • когда принят на работу

    Если сотрудник уволен:

    • ФИО сотрудника
    • уволен/работает
    • когда принят на работу
    • когда уволен

    4.4 ScheduleCard

    Отображает:

    • день недели
    • этаж уборки
    • сотрудник на этаже
